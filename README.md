# Avena Hyper-Bot: Polyglot Microservice Discord Bot

Добро пожаловать в репозиторий Avena Hyper-Bot! Это высокопроизводительный, многоязычный Discord-бот, разработанный с использованием микросервисной архитектуры для обслуживания тысяч серверов Discord с минимальным потреблением ресурсов.

## Архитектура

Бот построен на основе легковесного брокера сообщений NATS, который обеспечивает взаимодействие между узкоспециализированными микросервисами, написанными на разных языках программирования (Rust, Go, Python, C++, Zig, Bun, Lua). Каждый сервис выполняет свою задачу максимально эффективно, используя сильные стороны выбранного языка.

## Начало работы

Для запуска Avena Hyper-Bot вам потребуется Docker и Docker Compose, установленные на вашем сервере (например, Zotac).

### 1. Получение Discord Token

1.  Перейдите на [Discord Developer Portal](https://discord.com/developers/applications).
2.  Выберите ваше приложение (или создайте новое).
3.  В разделе **Bot** нажмите **Reset Token** и скопируйте полученный токен.

**ВНИМАНИЕ:** Discord Token является конфиденциальной информацией. Никогда не делитесь им и не выкладывайте в публичный доступ!

### 2. Настройка переменных окружения

Создайте файл `.env` в корневой директории вашего репозитория (рядом с `docker-compose.yml`) со следующим содержимым:

```dotenv
DISCORD_TOKEN="ВАШ_DISCORD_ТОКЕН_ЗДЕСЬ"
```

Замените `ВАШ_DISCORD_ТОКЕН_ЗДЕСЬ` на токен, который вы получили на Discord Developer Portal.

### 3. Запуск бота

1.  Клонируйте репозиторий на ваш сервер:
    ```bash
    git clone https://github.com/XIIIXXXIII/Avena.git
    cd Avena
    ```
2.  Убедитесь, что файл `.env` создан и содержит ваш Discord Token.
3.  Запустите Docker Compose:
    ```bash
    docker compose up --build -d
    ```
    *   `--build`: Пересобирает образы (необходимо при первом запуске или изменении кода).
    *   `-d`: Запускает контейнеры в фоновом режиме.

### 4. Проверка статуса

Вы можете проверить логи запущенных сервисов:

```bash
docker compose logs -f
```

Вы должны увидеть логи от `nats` и `gateway-watcher`. Если `gateway-watcher` успешно подключится к Discord, вы увидите сообщения о подключении и получении событий.

## Структура проекта

```
avena-hyper-bot/
├── docker-compose.yml       # Определение сервисов Docker Compose
├── .env                     # Переменные окружения (Discord Token)
├── LICENSE                  # Лицензия проекта (GPL v3)
├── README.md                # Этот файл
├── proto/                   # Protobuf схемы для обмена сообщениями
│   └── discord_events.proto
├── rust/                    # Сервисы на Rust
│   ├── Dockerfile
│   ├── Cargo.toml
│   └── src/
│       └── main.rs          # Gateway-Watcher (подключение к Discord)
├── go/                      # Сервисы на Go (placeholder)
├── python/                  # Сервисы на Python (placeholder)
├── cpp/                     # Сервисы на C++ (placeholder)
├── zig/                     # Сервисы на Zig (placeholder)
├── bun/                     # Сервисы на Bun (placeholder)
└── lua/                     # Сервисы на Lua (placeholder)
```

## Следующие шаги

После успешного запуска `gateway-watcher` мы сможем приступить к разработке других микросервисов (например, `command-router` на Go или `string-processor` на C++), которые будут подписываться на события из NATS и обрабатывать логику бота.
